# Обнаружение аномалий в ЭКГ-сигналах с помощью LSTM-автоэнкодера

## Описание задачи
Проект посвящен задаче детекции аномалий (Anomaly Detection) во временных рядах на основе датасета сердечных ритмов MIT-BIH. 

Вместо классического обучения с учителем (Supervised Learning), задача решена через подход Unsupervised Learning с использованием рекуррентного автоэнкодера. Модель обучается исключительно на нормальных кардиоциклах, учится их эффективно сжимать и восстанавливать. Предполагается, что на аномальных сигналах (патологиях) ошибка реконструкции модели будет значительно выше, что позволит зафиксировать аномалию.

 Архитектура LSTM Autoencoder (PyTorch)
 Данные Набор данных MIT-BIH Heartbeat Dataset ([Ссылка на источник на Kaggle](httpswww.kaggle.comdatasetsshayanfazeliheartbeat))

---

## Анализ данных и подготовка
1. Дисбаланс выборки В исходных данных наблюдается сильный дисбаланс (Нормальные семплы `90,589`, Аномальные семплы `18,857`). Данный факт дополнительно мотивирует использование подхода Anomaly Detection.
2. Логика разделения выборки Из нормальных данных сформированы обучающая (`train_df`) и валидационная (`val_df`) выборки. Тестовая выборка была сбалансирована и составлена из равного количества нормальных и аномальных сигналов для честной оценки качества классификации.

---

## Архитектура модели (LSTM Autoencoder)

Реализована рекуррентная нейросеть, состоящая из двух ключевых блоков
 Encoder Последовательно обрабатывает временной ряд (187 отсчетов) с помощью двух слоев `nn.LSTM`, сжимая его в фиксированный вектор скрытого пространства (`embedding_dim=64`).
 Decoder Разворачивает скрытое представление обратно в последовательность длины 187, пытаясь максимально точно восстановить исходный сигнал.
 Loss Function В качестве функции потерь и метрики оценки качества реконструкции использовался `L1Loss (MAE)`.

---

## Результаты и процесс обучения

Модель обучалась на протяжении 30 эпох с использованием оптимизатора `Adam` и начального `LR = 1e-3`. Ошибка на валидации стабильно падала вместе с ошибкой на обучении, достигнув финального значения `MAE ~ 5.6`.

### Оценка качества детекции
Был установлен жесткий порог ошибки реконструкции `THRESHOLD = 7`. Если ошибка восстановления сигнала превышает это значение, система сигнализирует об аномалии.

 Результат на тесте Из 4485 поданных аномальных кардиоциклов модель успешно зафиксировала 2794 (`~62.3% Recall`).

### Анализ ошибок по типам патологий
При анализе плотности распределения ошибок (`KDE Plot`) по конкретным подклассам аномалий были сделаны следующие выводы
1. Успешно детектируются Классы `Ventricular (V)` и `Fusion (F)`. Их графики ошибок значительно смещены вправо относительно порога, модель легко классифицирует их как патологию.
2. Сложные для модели классы `Supraventricular (S)` и `Unknown (Q)`. Их распределение ошибок сильно пересекается с распределением нормальных сигналов, из-за чего текущий базовый автоэнкодер их пропускает.

## Выводы и точки роста (Future Work)
Выбранный подход доказал свою применимость для явных анатомических аномалий сердца (классы V и F). Однако базовая архитектура оказалась недостаточно чувствительна к скрытым аномалиям (классы S и Q). 

Направления для улучшения проекта
 Протестировать более глубокую архитектуру (увеличить количество слоев LSTM или размер эмбеддинга).
 Попробовать сверточные автоэнкодеры (1D-CNN) или архитектуры на основе Transformer для лучшего извлечения локальных признаков сигнала.
 Применить адаптивный порог детекции, зависящий от конкретных статистических свойств сигнала.
